Automated Indian license plate detection and optical character recognition (OCR) system. Processes images, video files, and live webcam streams to detect vehicles, localize and read number plates, and log results to a SQLite database.

---

## Purpose

Designed for Indian traffic monitoring and vehicle identification. The system combines YOLOv8 object detection with EasyOCR to detect vehicles in a scene, crop the region where a license plate is expected (lower half of the vehicle), read the plate text, validate it against Indian plate formats, and log the detection with a snapshot.

**Intended users:** developers, security/traffic system integrators, and researchers working on ANPR (Automatic Number Plate Recognition) for Indian vehicles.

---

## Features

- **Three input modes:** single image, video file, and live webcam stream
- **Vehicle detection:** YOLOv8n (nano) filters for car, motorcycle, bus, and truck classes (COCO classes 2, 3, 5, 7)
- **Plate localization:** crops the lower 50% of each detected vehicle bounding box where plates are typically positioned
- **OCR text extraction:** EasyOCR with English language model
- **Indian plate format validation:** position-aware character correction (e.g., digit `0` at a letter position becomes `O`, letter `O` at a digit position becomes `0`)
- **Multi-frame consensus:** requires the same plate to be detected across multiple frames before logging, reducing false positives
- **Duplicate prevention:** time-window deduplication (10-second default) prevents re-logging the same vehicle
- **Image clarity scoring:** Laplacian variance used to select the sharpest capture of a plate
- **SQLite logging:** all detections stored in `database/vehicles.db` with timestamps, confidence scores, and snapshot paths
- **Snapshot capture:** saves annotated images and cropped plate images to `snapshots/` and `output/`
- **Batch evaluation:** `evaluate.py` processes all videos in `test_videos/` and generates performance reports
- **Image testing:** `image_test.py` processes all images in `test_images/` with per-image OCR logging
- **Auto-demo mode:** generates a synthetic vehicle image with a license plate when no input source is available
- **GPU/CPU auto-detection:** automatically uses CUDA if available, with manual override via `--gpu` / `--cpu`
- **Interactive mode:** menu-driven input selection when no CLI arguments are provided
- **Mouse/navigation not required:** fully keyboard-driven with `q` to quit

### Data Flow

1. **Input acquisition:** frames come from an image file (`cv2.imread`), video file (`cv2.VideoCapture`), or webcam (`cv2.VideoCapture`). Every 5th frame is processed (video/webcam).
2. **Vehicle detection:** YOLOv8n runs on the frame. Only detections with COCO class 2 (car), 3 (motorcycle), 5 (bus), or 7 (truck) are kept. Detection confidence threshold defaults to 0.5.
3. **Plate region crop:** for each vehicle, the lower 50% of the bounding box is extracted as the region where license plates are expected.
4. **OCR:** EasyOCR runs on each cropped region. Results with confidence > 0.3 and cleaned text length ≥ 4 are considered.
5. **Text validation:** `OCREngine.validate_indian_plate()` applies position-aware character corrections (`2`→`Z`, `0`→`O` at letter positions; `Z`→`2`, `O`→`0` at digit positions) and validates against Indian plate patterns (AA00AA0000, AA00AAA0000, AA0000).
6. **Consensus:** `PlateDetector.add_to_consensus()` tracks each normalized plate across frames. A plate is only captured after appearing in ≥2 frames (`min_consensus_frames = 2`).
7. **Capture & log:** once consensus is reached, the best-quality frame (highest OCR confidence) is saved as a snapshot to `snapshots/` or `output/`. The detection is logged to the `vehicles_log` table via `DatabaseManager.log_detection()`.
8. **Duplicate prevention:** `log_vehicle()` checks a recent-detection time window (10 seconds) before inserting into the database.

## How It Works

1. **Startup:** On launch, `main.py` detects the compute device (CUDA GPU or CPU), initializes the SQLite database (`DatabaseManager`), creates a `PlateDetector` instance, and loads the YOLOv8n model (from `models/yolov8n.pt` or downloaded on first use) and EasyOCR.

2. **Input selection:** If `--auto` is passed or no argument is given, the system tries sources in priority order: webcam → file in `input/` folder → synthetic demo image. The interactive mode presents a menu for manual selection.

3. **Frame processing (video/webcam):** Frames are read in a loop. Every Nth frame (5 for video, 10 for webcam) is passed to `detect_plates()`. YOLOv8 returns bounding boxes for all detected objects; only vehicle classes (car, motorcycle, bus, truck) are kept.

4. **Plate extraction:** For each vehicle bounding box, the lower half is cropped and sent to `extract_text()`. EasyOCR returns raw text with confidence scores.

5. **Text cleaning:** `OCREngine.clean_text()` strips non-alphanumeric characters, applies position-aware corrections (e.g., digit `0` in a letter position becomes `O`), and validates the result against standard Indian plate formats (10-character `AA00AA0000`, 11-character `AA00AAA0000`, 6-character old format `AA0000`).

6. **Consensus:** The normalized plate text is added to `frame_consensus`. Each unique plate must appear in at least 2 frames before being accepted. The frame with the highest OCR confidence is stored as a pending capture.

7. **Capture & log:** Once consensus is reached, the best pending frame is saved as a JPEG snapshot. `log_vehicle()` checks a 10-second duplicate window (preventing re-logging the same plate within that period), then inserts a row into `vehicles_log`.

8. **Display (video/webcam):** The current frame is annotated with bounding boxes and detected plate text, displayed in a named OpenCV window. The user can press `q` to stop.

9. **Single image mode:** Steps 3-5 are performed once. Results are printed to the console and the annotated image is saved to `output/`.

10. **Shutdown:** On completion or interruption, resources are released (`cap.release()`, `cv2.destroyAllWindows()`) and a summary is printed.
